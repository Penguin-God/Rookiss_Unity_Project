이번에는 XLM 안쓰고 Json 사용함. 에디터에서 버튼 하나 누르면 모든 엑셀 데이터가 Json으로 변환되도록 만듬. 그리고 런타임 중에는 Json만 사용.
엑셀 데이터를 관리할 때는 DB 정규화 하듯이 여러 데이터를 나누고, ID를 통해서 연결시킴.
레벨 별 데이터는 List로 저장함. 나중에 ID를 통해 다른 파일과 결합.

게임 매니저는 무기의 ID만 들고 있음. 세부 클래스는 데이터 매니저에 ID로 접근해서 사용함.
현재 컬랜디 게임 매니저는 전투 쪽에 너무 많이 결합되어 있음. 그래서 게임 매니저라고 부르기 애매함. 
이 프로젝트에서는 전투 씬을 관리하는 곳에서 게임 관련 로직들이 들어가 있음.

강화 게이지는 슬라이더로 관리. Base_UI에 따로 BindSlider()도 만듬.


인벤토리를 더 세부적으로 뜯어보기.
클라이언트 매니저를 이 프로젝트에서의 게임 매니저처럼 사용해야 함.
게임의 저장과 관련된 모든 데이터를 들고 있으면 될 듯. 점수나 가지고 있는 아이템 정보, 재화 같은 것들.
게임을 처음할 때는 Load()가 안되는 경우. 이때는 기본 스킬 주고 게임 시작.

핵심 : 레벨과 같이 한 무기가 여러 값을 수 있는 데이터와 이미지같은 고유 데이터를 구분한다.

루키스는 전역 변수를 만들어서 SetInfo()를 할 때 진짜로 정보만 저장함. 그리고 후에 Start 에서 실제로 UI를 그림.
나는 전역변수 만드는거 싫어하니까 SetInfo()에서 그리는거까지 바로 하려고 하는데, 그러면 Start에서 Bind가 안 된 상태로 작업을 하니까 꼼수를 써야하는 거임.
전역 변수를 만들던가 그리는 코드를 지저분하게 하든가 둘 중 하나임. 어차피 Init 검사하는 코드는 비슷하기는 하니까 그냥 할 듯?
